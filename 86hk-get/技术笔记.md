# 86hk.vip 商品数据抓取 — 技术笔记

## 概述

从号易系统（86hk.vip）后台抓取约 300 个在售商品的列表和详情数据，用于视频创作素材。

## 环境

- **站点**: https://www.86hk.vip （号易系统，基于 FastAdmin）
- **后台入口**: `/admin/dashboard?ref=addtabs`
- **工具**: Playwright MCP（Chromium 自动化浏览器）

## 登录与持久化

`.mcp.json` 配置 `--user-data-dir=.browser-profile`，浏览器 profile 保存在项目目录。首次手动登录后，Cookies/Session 自动持久化，后续无需重新登录。

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest", "--user-data-dir=.browser-profile"]
    }
  }
}
```

## API 端点

### 商品列表
```
GET /admin/goods/index?status=1&settlement_method=2&sort=id&order=desc&offset=0&limit=300&filter={}&op={}
```
- 返回 JSON，`total` 为总数，`rows` 为商品数组
- **必须带请求头** `X-Requested-With: XMLHttpRequest`，否则返回登录页 HTML

### 商品详情（"查看资料"）
```
GET /admin/goods/get_document?ids={parent_goods_id}
```
- `ids` 参数是 `parent_goods_id`（非商品 `id`）
- 返回 HTML 页面片段，结构为 `.copy_title` > `.f2.p`(标题) + `.f3/.f5/.f6`(内容)

## 抓取策略

### 关键坑：iframe + 并发 AJAX

后台页面采用 iframe 嵌套结构。在 iframe 内：
- **串行 AJAX 正常** — `await $.get()` 逐条请求，结果正确
- **并发 AJAX 失败** — `Promise.all()` 批量请求时，jQuery AJAX 返回空 HTML，解析结果全空

原因未深究，疑与 iframe 内 jQuery 的连接池或浏览器同源限制有关。**实际采用串行策略**，298 条约 1-2 分钟完成。

### 解析方式

在 iframe 内用 jQuery 解析 HTML：

```javascript
const $temp = $('<div>').html(html);
$temp.find('.copy_title').each(function() {
  const title = $(this).find('.f2.p').text().trim();      // 小标题
  const content = $(this).find('.f3, .f5, .f6');          // 内容区
  // .f3 = 红色背景框, .f5 = 带边框框, .f6 = 灰色背景框
  // 有 <p> 标签时按段落提取，用 \n 连接
});
```

注意：不能用主页面 `fetch()` + `DOMParser`，因为主页面请求不带 iframe 内的认证状态。

## 输出文件

| 文件 | 说明 |
|------|------|
| `86hk_products_full_v2.json` | **主文件**：列表 + 详情合并，291/298 条有完整数据 |
| `86hk_products_list.json` | 商品列表（名称、佣金、标签、结算规则） |
| `86hk_products_detail_v4.json` | 商品详情（14 个小标题 → 内容映射，含 text 和 html） |
