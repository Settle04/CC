# 并行master-worker架构的实现想法

## 目标
实现真正的多牛马并行：多个牛马实例同时执行任务，执行结果反馈到.md文件，走狗主动轮询审查。

## 架构图

```
用户（将军）
      ↓
  ┌───┴───┐
  │       │
牛马A   牛马B   ← 真正同时执行，互不等待
  │       │
结果.md 结果.md   ← 各自回写结果
  │       │
  └───┬───┘
      ↓
  用户统一回收 → 分发给走狗审查
```

## 核心机制

1. **牛马结果写入**：每个牛马实例执行完后，将结果写入独立的.md文件
2. **结果汇总**：用户统一回收各牛马的输出
3. **走狗审查触发**：用户下发命令让走狗查看结果

## 关键技术点

### 如何让走狗主动去看反馈？

方案1：**轮询**
- 牛马把结果写到固定位置（如 `results/A-任务.md`）
- 用 `/loop` 设定定时任务轮询 `results/` 目录
- 发现新文件就处理

方案2：**共享队列**
- 牛马写完结果后，往 `done_queue.md` 追加标记
- 走狗检测到标记就行动

方案3：**MCP通知机制**
- 如果有相关MCP支持文件系统监听，可实现实时触发

## Claude Code 内置方案（已验证可用）

### 方案A：Coordinator 模式

环境变量开启，会话级别生效：

```bash
export CLAUDE_CODE_COORDINATOR_MODE=1
claude
```

开启后进入协调器模式：
- **协调器（走狗）**：只负责理解意图、分解任务、派发Worker、综合结果
- **Worker（牛马）**：被协调器派发，独立执行任务
- 协调器不执行文件操作，只调用三个工具：`AgentTool`、`SendMessageTool`、`TaskStopTool`

### 方案B：Agent Teams（实验性）

```bash
--agent-teams
# 或
CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
```

官方描述：多Claude实例在 tmux/iTerm 分屏中并行工作。
