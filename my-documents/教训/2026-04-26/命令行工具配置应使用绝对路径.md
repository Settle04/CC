# 命令行工具配置应使用绝对路径

## 来源问题
- [MiniMax MCP 无法连接](../../问题集/2026-04-26/MiniMax%20MCP%20无法连接.md)

## 教训内容

配置 MCP Server 或其他命令行工具时，如果使用 `command` 字段指定可执行文件，**应使用绝对路径而非仅依赖 PATH 环境变量**。

### 错误做法
```json
{
  "command": "uvx"
}
```

### 正确做法
```json
{
  "command": "/home/settle/.local/bin/uvx"
}
```

### 为什么
- 不同环境下 PATH 可能不同
- Claude Code 等工具的 MCP 配置可能无法正确解析 PATH
- 使用绝对路径确保在任何情况下都能找到可执行文件

## 适用场景

- 配置任何 MCP Server 的 `command` 字段
- 配置 IDE/工具的外部命令路径时
- 任何需要指定可执行文件路径的场景
