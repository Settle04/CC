# 技术问题应先查官方文档

## 教训

遇到技术问题时，应该优先查阅官方技术文档，而不是靠猜测和试错。

## 背景

在解决 Playwright 登录状态持久化问题时，错误地尝试了多种基于 cookies 的方案，浪费了大量时间。实际上 Playwright 官方明确推荐使用 `storageState` API 来解决这个问题。

## 问题分析

- **错误做法：** 猜测 Playwright 的实现方式，尝试手动保存 cookies
- **原因：** 没有先查阅官方文档
- **后果：** 多次迭代失败，耗时过长

## 正确做法

1. **遇到技术问题 → 先查官方文档**
2. **确认官方推荐方案 → 再实施**
3. **官方文档不清晰 → 再尝试社区方案**

## Playwright 正确方案参考

官方文档明确说明使用 `storageState`：

```javascript
// 登录时保存状态
const context = await browser.newContext();
await context.storageState(); // 自动保存完整状态

// 加载状态
const context = await browser.newContext({
  storageState: 'state.json'  // 自动加载已有状态
});
```
